Chapter I.

Nature and design of this work

Hy is a project to compile a Lisp variant down to Python AST. It does this by being just way too clever for its own good. Hy is a front-end (not unlike gcc or clang) that parses, lexes and compiles S-Expressions into an IR (in our case, Python AST.)

There are a number of core rules you must keep in mind when working with Hy. The most fundamental is that Hy compiles to Python, directly. This means everything is always bidirectionally compatible, which means you can do such crazy things as write Django apps in Hy.

  1. Dashes turn to underscores
  2. So called “earmuffs” get converted to uppercase
  3. Remember that you have all of Python's stdlib and module ecosystem at your fingertips

Hy code should be runnable on Python version 2.6, 2.7, 3.2, or 3.3 as well as pypy. It might need some small changes (bytes vs strings, some functions return generators in Python 3), but most things work great.

Chapter II.

“How to get Hy really fast”

Getting started with Hy is a snap; just install virtualenv, activate the virtualenv and go ahead and pip install hy. Now the hy(1) command is ready for REPL-ication and general tomfoolery.

$ hy
=> (+ 1 2 (* 2 3))
9
